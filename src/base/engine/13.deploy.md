# 十三.部署上线

::: tip 场景
前端开发需要确认已发布，测试是否成功发布本次项目代码
:::

## 1.部署前

::: warning 注意
有时候部署比较敏感，如上 uat、prod 环境，为了防止出意外导致投诉（如测试或领导），可以先在本地模拟线上环境，校验前端打包后的代码是否是正常的，自己确定修改无误后再部署
:::

- 本地打包代码
- 本地启用 nginx，验证打包代码

  nginx/conf/nginx.conf

  ```js
  ...
  location /[项目二级目录] {
    root D:/project/[项目名称]
    try_fiels $uri $uri/ /[项目二级目录]/index.html
    index index.html index.htm
  }
  ...
  ```

## 2.部署

- 直接上传打包代码到服务器
- 通过相关工具部署流水线

## 3.部署后

- 查看代码是否更新
